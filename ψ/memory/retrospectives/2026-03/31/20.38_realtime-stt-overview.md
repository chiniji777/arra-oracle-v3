# Session Retrospective — Real-time STT for Overview Voice Popup

## Session Summary

เปลี่ยน voice transcription ใน Overview page จาก **MediaRecorder → Gemini API** เป็น **Browser SpeechRecognition** แบบ real-time ตัวหนังสือวิ่งเข้า textarea ทันทีขณะพูด

## What Got Done

### 1. ลบ Gemini จาก STT flow
- ลบ EnhancedSR wrapper ใน `dist-office/index.html` (เดิม wrap SpeechRecognition แล้วส่งเสียงไป Gemini refine อีกรอบ)
- ลบ `/api/tts` endpoint จาก `cli.ts` (Gemini TTS — ไม่ใช้แล้ว)
- `/api/transcribe` ยังอยู่แต่ถูก intercept ไม่เรียก Gemini จริง

### 2. Inject real-time SpeechRecognition ใน index.html
- MutationObserver จับ `textarea[placeholder*="Speak"]` เมื่อ voice popup เปิด
- Start `SpeechRecognition` with `interimResults: true, continuous: true, lang: "th-TH"`
- `onresult` เขียน final + interim text เข้า textarea ทันทีผ่าน native value setter + React input event
- Track `finalCount` เพื่อไม่ให้ text ซ้ำเมื่อ results สะสม

### 3. Mock MediaRecorder + getUserMedia
- Mock `navigator.mediaDevices.getUserMedia` → return silent AudioContext stream
- Mock `MediaRecorder` → no-op (ไม่อัดเสียงจริง)
- ป้องกัน MediaRecorder แย่ง mic จาก SpeechRecognition

### 4. UI tweaks
- ซ่อน gray interim text ใต้ textarea ด้วย CSS (`textarea ~ div { display: none }`)
- ขยาย textarea (min-height 80px, max-height 200px)

### 5. Debug & fixes
- แก้ MutationObserver error "parameter 1 is not of type Node" — ย้าย script จาก `<head>` ไปหลัง `<body>`
- แก้ text ซ้ำ — ใช้ `finalCount` track finals ที่เคยเห็น
- แก้ mic ไม่เข้า — ค้นพบว่า USB mic device ไม่ทำงาน สลับไปใช้ MacBook Pro Built-in mic

## Lessons Learned

1. **Script ใน `<head>` ไม่เห็น `document.body`** — MutationObserver ที่ observe body ต้องรอ DOM ready หรือวาง script หลัง body
2. **SpeechRecognition results สะสม** — `ev.results` มี final เก่าอยู่ด้วย ต้อง track count ไม่งั้น text ซ้ำ
3. **MediaRecorder + SpeechRecognition แย่ง mic** — ต้อง mock getUserMedia ให้ return silent stream เพื่อให้ SpeechRecognition ได้ mic exclusive
4. **React textarea control** — ใช้ native value setter + dispatch `input` event ถึงจะ sync กับ React state ได้
5. **Monkey-patch มี limit** — ยิ่ง patch ซับซ้อน ยิ่งมี side effect ถ้าต้องทำอะไรมากกว่านี้ควรแก้ source แล้ว build ใหม่
