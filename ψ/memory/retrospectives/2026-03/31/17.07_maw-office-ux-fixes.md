## Session Summary

Continued from previous session (voice STT/TTS work). This session focused on UX fixes and improvements to the maw-js Office frontend — specifically the Overview page's Live Terminal popup (HoverPreviewCard) and the fullscreen XTerminal component. All changes were done by patching the built JS (`dist-office/`) since the frontend can't be rebuilt from source.

## What Got Done

### 1. Removed TTS System
- Deleted the entire TTS `<script>` block from `dist-office/index.html` (~255 lines)
- Removed all: TTS state manager, speak functions (browser + gemini), status-based polling, UI buttons, engine selector
- STT system (Web Speech + Gemini refinement) kept intact

### 2. Fixed File Drag-Drop Path Not Included
- **Root cause**: `HoverPreviewCard.sendMessage` joined text + file paths with `\n` (newline). When pasted via tmux `paste-buffer`, the `\n` acts as Enter — submitting the text before the file path.
- **Fix**: Changed `.join("\n")` to `.join(" ")` — text and paths now on same line
- Also fixed `XTerminal.handleFiles` to add visual feedback (`[uploading...]` / `[attached → path]`) and trailing space, matching the paste handler behavior

### 3. Quick-Action Button Changes
- Replaced `/help` button with `/compact` (more useful for daily workflow)
- Removed `exit` button (dangerous, not needed)

### 4. Fixed Screenshot Paste in Popup
- **Root cause**: Paste handler checked `document.activeElement !== inputRef.current` — only worked when textarea was focused
- **Fix**: Removed the focus check — paste now works from anywhere when the card is open

## Lessons Learned

- **Monkey-patching built JS is fragile but effective**: Using Python regex to find/replace minified patterns works when you can't rebuild. Always verify the match count is exactly 1 before replacing.
- **tmux `paste-buffer` treats `\n` as Enter**: When sending multi-part text via tmux, use spaces not newlines to keep it as one message. `sendText()` uses `load-buffer` + `paste-buffer` for multiline content.
- **Trace the full send chain**: HoverPreviewCard → WebSocket `{type:"send"}` → `handlers.ts:send` → `ssh.ts:sendKeys` → `tmux.ts:sendText` → `load-buffer` + `paste-buffer` + Enter. Understanding each layer was essential to finding the real bug.
