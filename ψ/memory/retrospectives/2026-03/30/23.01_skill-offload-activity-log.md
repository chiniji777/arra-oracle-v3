## Lessons Learned

1. **Skills คือตัวกิน context หลัก** — 36 skills = 230KB กิน ~10-15% context ทันทีตอน spawn ทั้งที่ไม่ได้ใช้ทุกตัว
2. **Skill router pattern ได้ผล** — ให้ agent 1 ตัว (Saraswati) เป็น knowledge hub แทนที่ทุกคนจะแบกทุก skill
3. **2 Phase approach ลดความเสี่ยง** — Phase 1 ใช้กฎใน CLAUDE.md ก่อน พิสูจน์ว่า useful แล้วค่อย automate ใน Phase 2
4. **Cross-agent review ดีมาก** — ส่งแผนให้ Saraswati วิเคราะห์ก่อน deploy ช่วยจับจุดที่ FirstGod มองข้าม (path กลาง, atomic write)
5. **ถามก่อนทำ ดีกว่าทำก่อนถาม** — Nut สั่งไม่ให้ Saraswati ตรวจ compliance ให้ FirstGod ทำเอง + ไม่ให้ agent โหลด skill ทุกตัว ถ้าไม่ถามก่อนจะทำผิดทิศ
